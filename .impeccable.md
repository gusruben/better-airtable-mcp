# Better Airtable MCP — design context

## Design Context

### Users
Augie and Hack Club staff who receive an approval link when an AI agent (Claude) wants to
create/update/delete records in an Airtable base through this MCP. They open the link, review the
exact change (a record diff), and approve or reject it. Often opened mid-task, sometimes on mobile.
A secondary surface is the `/debug` page — an internal tool for exercising MCP sessions/tools.

### Brand Personality
Airtable-native: clean, calm, dense, trustworthy. This is not a place for personality or flourish —
it should feel like a built-in Airtable extension panel so the reviewer instantly trusts it and can
scan the change quickly. Voice: plain, direct, reassuring ("Waiting for your approval", not
"pending_approval").

### Aesthetic Direction
Match Airtable's product UI exactly. **Source of truth: the real Airtable CSS bundle**, captured
from the live app via browser traffic and saved in `claude-workspace/airtable-ref/css/`
(`static.airtable.com/.../all.*.css`). The Figma kit was scrapped — it was an older/approximate
look. Real tokens: **Inter** typeface (13px base), primary **`#166EE1`** (hover `#0D52AC`), text
`#1D1F25`, cool blue-tinted grays (`#F6F8FC`/`#F2F4F8`/`#E5E9F0`), translucent borders
`rgba(0,0,0,0.1)`, 3px control / 6px surface radii, subtle `--elevation-low` shadows, focus ring
`0 0 0 2px #166EE1`. Per-component fidelity (not just tokens): update diffs use Airtable's
revision-history style (`#f9f9f9` chips, removed value strikethrough in `#ED5A23`); multi-select →
22px pill choice tokens; linked records → `#deebff` chips; grid-density tables.

**No base icon tile (intentional).** The header shows the base name as a plain link to the base
in Airtable — no colored icon tile. The public Meta API (`/v0/meta/bases`) returns only
id/name/permissionLevel and does NOT expose a base's real color or icon, so any tile color would
be a guess. Decision (2026-06-26): don't fake it. Don't re-add a derived-color base tile.

### Design Principles
1. **Fidelity over novelty.** `tokens.md` is the authoritative source of truth and **overrides
   impeccable's defaults** — do NOT pick a "distinctive" font, do NOT avoid the chosen palette, do
   NOT build bold asymmetric layouts. The brief is "look like Airtable," not "look unique."
2. **Flat, not floating.** Borders define structure, not shadows/blur. 3px radii everywhere
   (pills only for tokens/toggles).
3. **Dense and legible.** 13px base UI text, tight Airtable cell density, generous whitespace
   between groups (4pt scale).
4. **Plain language.** No raw enums or jargon in the UI; every status/label reads like a sentence.
5. **Every state covered.** Loading, error, empty, and all 7 operation statuses get a clear,
   styled state — never a silent empty or a raw backend error dump.
